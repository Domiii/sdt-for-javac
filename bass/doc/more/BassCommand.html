BASS -- Bytecode ASSembly language assembler

Bass is an assembler to translate bytecode assembly program (.bass file) to the corresponding Java bytecode (.class file).

Contents

  • Synopsis
  • Options
  • Examples

Synopsis

bass [ options ] bassFiles

Arguments may be in any order.

options
Command-line options.

bassFiles
One or more source files to be assembled.

Options

-cp classpathlist
Specify a list of directories, JAR archives, and ZIP archives where bass will look for referenced classes. Class path entries are separated by semicolons (;). The pre-defined class path consists of the current directory and the jdk boot class path. If -cp is specified, classpathlist is used before the pre-defined class path.

-d directory
Set the destination directory for class files. The value directory can be absolute or relative to the current working directory. If a class is part of a package, bass puts the class file in a subdirectory reflecting the package name, creating directories as needed. For example, if you specify -d c:\Bass\classes and the class is called lab441.bass.Main, then the class file is called c:\Bass\classes\lab441\bass\Main.class. If -d is not specified, the destination directory is the current working directory.

Examples

bass String.bass reflect\Field.bass

bass -cp jdk\lib\tools.jar;myjavac\classes -d myjavac\classes *.bass
